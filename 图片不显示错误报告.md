# 图片不显示错误报告

## 1. 问题概述

- 问题现象：前端页面书籍封面无法显示。
- 典型报错：
  - `GET /media/book_covers/2809.png 500 (Internal Server Error)`
  - `GET /media/book_covers/2809.png 404 (Not Found)`
  - `net::ERR_CONNECTION_REFUSED`（请求发到错误地址或服务未监听）
- 影响范围：用户端、商家端、管理端中所有依赖书籍封面的页面。

## 2. 根因分析

本次问题并非单一原因，而是多个因素叠加：

1. 运行目录错误导致命令执行失败

- 在 `openbookshop/Library` 目录执行 `python manage.py runserver 8000`，该目录不存在 `manage.py`，导致后端未正确启动。

2. Docker 环境下媒体路由未生效

- 系统在 Docker 中使用 `DJANGO_DEBUG=False`。
- 原先通过 `django.conf.urls.static.static(...)` 添加媒体路由，但该方法在 `DEBUG=False` 时会返回空路由。
- 结果：`/media/book_covers/...` 请求进入 Django 后返回 404。

3. 图片 URL 生成方式在反向代理下不稳定

- 业务代码中使用 `request.build_absolute_uri(...)` 生成绝对 URL。
- 在 Nginx/容器代理场景下，绝对 URL 可能出现主机名或端口不符合前端访问环境的问题。
- 导致图片请求地址不稳定，增加失败概率。

## 3. 处理方法

### 3.1 规范运行命令与目录

- 正确目录：`E:\opencode\openbookshop\openbookshop\backend`
- 正确命令：

```bash
python manage.py runserver 8000
```

### 3.2 修复 Docker 下媒体文件路由

在 Django 路由中显式添加媒体与静态路由，避免依赖 `DEBUG=True`。

修改文件：

- [openbookshop/backend/config/urls.py](openbookshop/backend/config/urls.py)

核心修改：

```python
from django.urls import include, path, re_path
from django.views.static import serve

urlpatterns += [
    re_path(r'^media/(?P<path>.*)$', serve, {'document_root': settings.MEDIA_ROOT}),
    re_path(r'^static/(?P<path>.*)$', serve, {'document_root': settings.STATIC_ROOT}),
]
```

### 3.3 调整封面 URL 返回策略为相对路径

将封面 URL 返回从“绝对路径”改为“相对路径”，由前端/Nginx统一转发，更适合容器化部署。

修改文件：

- [openbookshop/backend/apps/books/serializers.py](openbookshop/backend/apps/books/serializers.py)
- [openbookshop/backend/apps/orders/views.py](openbookshop/backend/apps/orders/views.py)

核心思路：

- 旧逻辑：`request.build_absolute_uri(obj.cover.url)`
- 新逻辑：直接返回 `obj.cover.url`（例如 `/media/book_covers/2809.png`）

## 4. 验证结果

已执行容器重建与接口验证：

1. 重新构建并启动后端

```bash
docker-compose up -d --build backend
```

2. 验证后端直连

```bash
curl -I http://127.0.0.1:8000/media/book_covers/2809.png
```

返回：`HTTP/1.1 200 OK`

3. 验证前端经 Nginx 访问

```bash
curl -I http://127.0.0.1:8080/media/book_covers/2809.png
```

返回：`HTTP/1.1 200 OK`

结论：图片访问链路恢复正常。

## 5. 经验总结与防复发建议

1. 开发与排障时先确认“启动目录 + 启动命令”是否正确。
2. Docker 生产化场景不要依赖 `DEBUG=True` 提供媒体路由。
3. 在反向代理体系中，优先返回相对资源路径，减少 Host/端口错配问题。
4. 每次修改后统一执行三段验证：
   - 后端端口直连验证（8000）
   - 前端入口验证（8080）
   - 浏览器 Network 面板验证（状态码与资源类型）
