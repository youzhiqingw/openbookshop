## 🚨 故障排查

### 登录不了?

**检查 1: 容器是否运行**
```bash
docker-compose ps
# 应该显示 3 个 running 容器
```

**检查 2: 用户是否存在**
```bash
docker exec openbookshop_backend python manage.py shell -c "
from apps.users.models import User
print(User.objects.count())  # 应该是 3
"
```

**检查 3: 认证是否成功**
```bash
docker exec openbookshop_backend python manage.py shell -c "
from django.contrib.auth import authenticate
user = authenticate(username='merchant_test', password='merchant123456')
print('认证成功' if user else '认证失败')
"
```

### 页面加载很慢?

- 检查 Docker 容器资源占用: `docker stats`
- 查看后端日志: `docker-compose logs backend`
- 查看前端日志: `docker-compose logs frontend`

### 数据丢失了?

> ⚠️ **重要**: 使用 `docker-compose down -v` 会删除所有数据！

正确的做法:
```bash
# 只停止容器，保留数据
docker-compose down

# 重启时数据不丢失
docker-compose up -d
```
